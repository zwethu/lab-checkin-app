# Smart Lab Check-in App

A Flutter web app for students to check in and check out of lab classes, track mood, log what they learned, and submit feedback — all synced in real time with Firebase.

---

## Project Description

Smart Lab Check-in is a mobile-first web application built with Flutter and Firebase. Students can:

- View today's lab classes fetched from Firestore
- **Check in** before class by entering their Student ID, previous topic, expected topic, and mood
- **Finish class** by logging what they learned and optionally submitting feedback
- View their full check-in **history** synced from the cloud
- GPS coordinates are recorded at both check-in and check-out

The app includes a **landing page** at `/landing` as a standalone info/ads page, and the main app lives at `/`.

## Setup Instructions

### Prerequisites

- Flutter SDK `3.24.3` or compatible
- Dart SDK `3.5.3`
- Firebase CLI installed (`npm install -g firebase-tools`)
- A Firebase project (already configured: `my-lab-checkin-app-79`)

### 1. Clone the repository

```bash
git clone https://github.com/zwethu/lab-checkin-app.git
cd lab-checkin-app
```

### 2. Install dependencies

```bash
flutter pub get
```

### 3. Generate Hive adapters (if not already generated)

```bash
dart run build_runner build --delete-conflicting-outputs
```

***

## How to Run

### Run on Chrome (Web)

```bash
flutter run -d chrome
```

### Run on Android

```bash
flutter run -d android
```

### Build and Deploy to Firebase Hosting

```bash
flutter build web && firebase deploy --only hosting
```

|---|---|

## Firebase Configuration Notes

- Firebase is already configured via `lib/firebase_options.dart` (generated by FlutterFire CLI)
- The Firestore database uses two collections:

### `classes` collection
Each document represents a lab class:
```
{
  id: string,
  name: string,
  time: string,
  room: string
}
```

### `checkins` collection
Each document represents one student check-in:
```
{
  classId: string,
  className: string,
  studentId: string,
  classDate: string (ISO 8601),
  checkinTime: string (ISO 8601),
  checkinLat: number,
  checkinLng: number,
  previousTopic: string,
  expectedTopic: string,
  moodBefore: number (1-5),
  finishTime: string | null,
  finishLat: number | null,
  finishLng: number | null,
  learnedToday: string | null,
  classFeedback: string | null
}
```

- Firestore data is loaded on app startup via `syncFromFirestore()` as the source of truth
- Hive is used as a local offline cache on **native platforms only** (Android/iOS)
- On **web**, state is driven entirely from Firestore

### Re-configuring Firebase (if needed)

```bash
flutterfire configure
```
